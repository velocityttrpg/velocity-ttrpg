# Skills

## 1. What a Skill Is

1. A skill is a discrete capability a character has learned and can bring to bear in play.
2. Skills are binary — a character either has a skill or does not. There are no partial investments or incremental upgrades within a single skill.
3. Each skill belongs to a skill tree. Trees represent a domain of training or a lineage path.
4. Skills are purchased using Skill Points. See **Skill Points** for how Skill Points are earned and spent.

## 2. How Skills Work

1. Each skill does exactly what its description says — no more, no less.
2. Skills stack. All applicable skill effects apply simultaneously.
3. A skill applies when the situation matches its description.
4. Skills do not have a universal effect. There is no rule that owning a skill automatically adds a die to related rolls — the skill's description defines its effect entirely.
5. A skill that does not apply to the current situation provides no benefit to that situation.

## 3. Skill Cost

1. Most skills cost 1 Skill Point to purchase.
2. Some skills cost more than 1 Skill Point. The increased cost is stated in the skill's description.
3. A character must meet all prerequisites before purchasing a skill, regardless of cost.

## 4. Nodes and Skills

1. Skill trees contain two types of elements: nodes and skills.
2. Nodes are structural organizers. They group related skills and gate access to the skills beneath them. Nodes do not require purchase.
3. Skills are purchasable capabilities. They require Skill Points to acquire and provide mechanical effects when purchased.
4. A character may access any skill beneath a node without purchasing the node itself.

## 5. Skill Trees and Prerequisites

1. Skills within a tree are organized from foundational to advanced.
2. A character must own a skill's parent skill before purchasing any skill that branches from it.
3. Some advanced skills have an additional investment threshold requirement — a minimum number of Skill Points spent within the same tree. This threshold is stated in the skill's description.
4. There is no requirement to purchase every skill in a tree. Characters may follow any path through the tree as long as prerequisites are met.

## 6. Using Skills

1. Skill Points are spent between sessions, outside of play.
2. When a skill is purchased, the character immediately gains all benefits described in that skill.
3. When a skill is lost through retraining, the character immediately loses all benefits and penalties granted by that skill. See **Skill Points** for retraining rules.

## 7. Skill Trees

1. Skill trees are organized into two categories: Universal and Setting.
2. Universal trees are available to every character regardless of lineage or campaign setting.

        7. [Medium Armor](Universal/Combat/Medium Armor.md)
        8. [Heavy Armor](Universal/Combat/Heavy Armor.md)
    2. **Influence** (purchasable trunk — Influence I required to unlock branches)
        1. [Influence](Universal/Influence/Influence.md)
        2. [Persuasion](Universal/Influence/Persuasion.md)
        3. [Intimidation](Universal/Influence/Intimidation.md)
        4. [Negotiation](Universal/Influence/Negotiation.md)
    3. **Survival** (node)
        1. [Athletics](Universal/Survival/Athletics.md)
        2. [Apothecary](Universal/Survival/Apothecary.md)
        3. [Stealth](Universal/Survival/Stealth.md)
        4. [Tracking](Universal/Survival/Tracking.md)
        5. [Wilderness](Universal/Survival/Wilderness.md)
    4. **Investigation** (purchasable trunk — Investigation I required to unlock branches)
        1. [Investigation](Universal/Investigation/Investigation.md)
        2. [Forensics](Universal/Investigation/Forensics.md)
        3. [Research](Universal/Investigation/Research.md)
        4. [Supernatural Detection](Universal/Investigation/Supernatural Detection.md)
        5. [Situational Awareness](Universal/Investigation/Situational Awareness.md)
    5. **Attributes** (node)
        1. **Social** (constellation)
            1. [Dominance](Universal/Attributes/Dominance.md)
            2. [Empathy](Universal/Attributes/Empathy.md)
        2. [Fortitude](Universal/Attributes/Fortitude.md)
        3. [Reflex](Universal/Attributes/Reflex.md)
        4. [Strength](Universal/Attributes/Strength.md)
        5. [Vitality](Universal/Attributes/Vitality.md)
        6. [Willpower](Universal/Attributes/Willpower.md)
3. Attribute skills have no prerequisites and are always available.
4. Setting trees are defined by the campaign setting and are not part of the core rules.
    1. Setting trees cover lineage paths and setting-specific capabilities.
    2. Campaign settings built on Velocity may define their own setting trees.
